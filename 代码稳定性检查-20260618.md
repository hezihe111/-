- `app.js` JavaScript 语法检查通过。
- 顶层函数无重复声明。
- HTML 固定节点与 JavaScript 直接 DOM 引用一致；检测到的交易选择框属于弹框动态节点，不是缺失元素。
- 弟子重大抉择已通过真实年度流程测试：年度战报 → 重大抉择 → 结果 → 年度天命，控制台无错误。

- 新增弟子事件年份、边境声望、补给、兽潮、区域事件和据点字段均加入默认状态与旧存档兼容初始化。
- 前端资源版本已更新，避免部署后继续命中旧 JS/CSS。

## 本轮稳定性优化

1. 重大路线唯一入口
   - 移除养成面板的主动转修、主动拜师、洗髓觉醒和手动个人事件按钮。
   - 底层旧函数增加拒绝保护，旧缓存按钮也不能绕过规则。

2. 年度事件队列
   - 弟子重大抉择作为正式队列节点执行，完成后才继续世界奇遇、宗门大比、仙盟、拍卖和天命。
   - 抉择弹窗与结果弹窗均不可直接关闭，避免回调丢失导致年度卡死。

3. 边境节点生命周期
   - 修正“新副本生成后立即扣寿命”的问题。
   - 旧副本和事件先跨年结算，再追加新内容。
   - 副本上限 10、区域事件上限 4，避免长期存档节点无限增长。

4. 存档兼容
   - 旧存档缺少新字段时自动补齐，不要求重开。
   - 联机共享的 `frontier` 对象包含新增边寨、补给、事件与兽潮状态。

5. 卡片布局
   - 使用原始卡框真实比例 `990×1515`。
   - 六个属性槽按原画像素中心定位，删除额外假槽。

## 后续架构建议

- 当前 `app.js` 已超过单文件适合维护的规模。下一阶段建议按 `state/save`、`disciples`、`frontier`、`combat`、`multiplayer`、`ui/modal` 六个模块拆分。
- 拆分前先建立自动化存档迁移测试与年度事件队列测试，避免纯粹移动代码产生回归。
- 联机消息仍以较多字符串分支处理，后续可增加统一消息 schema 校验和版本号。
- 地图节点类型建议改为注册表模式，让绘制、提示、操作和半径定义集中在一处，减少新增节点时漏改分支的风险。
